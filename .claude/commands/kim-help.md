
### 情况3：参数是需求描述 - 智能推荐命令

如果参数不匹配任何命令名，则视为需求描述，进行智能匹配推荐：

**匹配规则**：

| 关键词 | 推荐命令 | 原因 |
|--------|----------|------|
| 配置、环境、安装、setup | /kim-setup | 环境相关 |
| 帮助、命令、怎么用 | /kim-help | 使用指导 |
| 拆解、计划、任务、需求分析 | /kim-plan | 需求规划 |
| CRUD、增删改查、数据库 | /kim-crud | CRUD模板 |
| API、接口、端点、endpoint | /kim-api | API生成 |
| 表单、form、输入、校验 | /kim-form | 表单组件 |
| 生成、写代码、实现、开发 | /kim-code | 代码生成 |
| 审查、review、检查、质量 | /kim-review | 代码审查 |
| 完整、生产、正式、团队 | /kim-team | 完整流程 |

💡 **注意**：简单问题直接对话即可，无需使用命令

## 使用示例

```bash
# 查看所有命令
/kim-help

# 查看特定命令详情
/kim-help team
/kim-help code
/kim-help setup

# 根据需求获取推荐
/kim-help "我想生成一个用户表的CRUD"
/kim-help "帮我审查一下代码质量"
/kim-help "我有一个大需求，想拆成小任务"
```
